# Data Model: Autonomous GitHub Issue Fixer

**Phase**: 1 — Design
**Date**: 2026-04-08

## Entities

### Issue
Represents a GitHub issue fetched from the API.

| Field | Type | Description |
|-------|------|-------------|
| url | str | Full GitHub issue URL (input) |
| repo_owner | str | Parsed from URL |
| repo_name | str | Parsed from URL |
| number | int | Issue number |
| title | str | Issue title |
| body | str | Issue description |

### Repository
Represents the cloned repo in the local sandbox.

| Field | Type | Description |
|-------|------|-------------|
| full_name | str | owner/repo |
| default_branch | str | e.g. "main" |
| local_path | str | Absolute path to temp dir clone |
| has_tests | bool | Whether a test suite was detected |

### AgentRun
Represents one full execution of the agent loop.

| Field | Type | Description |
|-------|------|-------------|
| issue | Issue | The issue being fixed |
| repo | Repository | The cloned repo |
| attempts | int | Number of fix attempts made |
| tool_calls | int | Total tool calls made |
| status | str | "success" \| "failed" \| "gave_up" |
| files_changed | list[str] | Files modified by the agent |
| pr_url | str \| None | URL of opened PR if successful |

### Fix
Represents a single fix attempt within an AgentRun.

| Field | Type | Description |
|-------|------|-------------|
| attempt_number | int | Which attempt this is (1-indexed) |
| files_changed | list[str] | Files written in this attempt |
| test_output | str | Raw stdout/stderr from test run |
| tests_passed | bool | Whether tests passed after this fix |

### PullRequest
Represents the draft PR opened on success.

| Field | Type | Description |
|-------|------|-------------|
| title | str | Generated by Claude |
| body | str | Generated by Claude, references issue |
| branch | str | New branch created for the fix |
| url | str | GitHub PR URL after creation |

## State Transitions

```
AgentRun.status:
  RUNNING → success     (tests pass, PR opened)
  RUNNING → failed      (max retries hit, tests never passed)
  RUNNING → gave_up     (Claude calls finish with status=gave_up)
```

## Tool Call Schema

### list_directory
```json
{ "path": "string" }
→ returns: "file1.py\nfile2.py\nsubdir/\n..."
```

### read_file
```json
{ "path": "string" }
→ returns: file contents as string
```

### write_file
```json
{ "path": "string", "content": "string" }
→ returns: "ok" or error message
```

### run_tests
```json
{ "timeout_seconds": 60 }
→ returns: stdout + stderr from pytest/unittest
```

### finish
```json
{
  "status": "success | gave_up",
  "pr_title": "string",
  "pr_body": "string",
  "files_changed": ["string"],
  "reason": "string (if gave_up)"
}
→ terminates agent loop
```
